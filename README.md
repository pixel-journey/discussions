## Our Guiding Principles

All discussions here are expected to align with (or thoughtfully challenge) these core tenets:

- **Zero custom contract overhead** — maximize existing public primitives (AtomicAssets, Alcor Concentrated Liquidity, Hyperion, eosio.msig, etc.)
- **Client-side state & rendering** — prefer local processing, Web Workers, Canvas, and browser APIs over unnecessary backends
- **On-chain entropy for randomness** — use deterministic parsing of transaction hashes + block headers for provably fair mechanics (no oracles or RAM fees)
- **Educational clarity first** — every thread should help others learn and build
- **Retro-pixel soul + modern rigor** — beautiful, accessible interfaces with strict TypeScript and clean architecture
- **Transparent & auditable** — GitOps patterns, versioned data, and open reasoning are preferred

## Getting Involved

1. Browse active discussions in the categories above.
2. Start a new thread in the most relevant category.
3. For major new blueprint ideas, open a thread in **Blueprint Ideas & RFCs** first — this helps the community give early feedback before code is written.
4. Share wins and beautiful experiments in **Show & Tell**.
5. Help newcomers in **WAX Dev Q&A** and **Education & Onboarding**.

We especially value contributions that advance:
- Verifiable fairness and on-chain entropy patterns
- Generative creativity that feels native to WAX
- Accessible, high-quality educational tooling
- Clean, maintainable client-side architectures
